### Итоговое техническое задание (ТЗ)

Поскольку вы не предоставили ответы на уточняющие вопросы, я использовал значения по умолчанию для полноты. ТЗ структурировано для удобства разработчиков: оно охватывает все ключевые аспекты, исключает двусмысленности и ориентировано на MVP (минимально жизнеспособный продукт). Если нужны изменения, укажите детали для доработки.

#### 1. Общая информация
- **Название проекта**: FitTrack (по умолчанию; можно изменить).
- **Тип проекта**: Мобильное приложение для смартфонов.
- **Платформы**: iOS и Android (кроссплатформенная разработка).
- **Цель**: Создать фитнес-приложение с функциями для проверки техники упражнений через камеру, генерации диеты с трекером питания и отслеживания статистики прогресса.
- **Целевая аудитория**: Взрослые пользователи (18+), интересующиеся фитнесом, контролем веса и здоровым питанием (новички и опытные).
- **Языки**: Русский (основной; с возможностью добавления английского в будущем).
- **Монетизация**: Бесплатное приложение с потенциалом для рекламы или премиум-фич (по умолчанию: без монетизации в MVP).
- **Сроки разработки**: 3-4 месяца на MVP (ориентировочно, на основе плана для 2 человек).
- **Бюджет**: Не определен (по умолчанию: ориентирован на небольшую команду без внешних интеграций).

#### 2. Функциональные требования
Приложение состоит из трех основных функций. Все функции должны быть интегрированы в единый интерфейс с навигацией (например, bottom navigation bar: "Упражнения", "Диета", "Профиль").

##### 2.1. Функция 1: Проверка правильности выполнения упражнений
- **Описание**: Пользователь выбирает упражнение, активирует камеру, и приложение анализирует движения в реальном времени, предоставляя обратную связь о технике.
- **Ключевой функционал**:
  - Выбор упражнения: Список из 10 базовых упражнений (по умолчанию: приседания, отжимания, планка, подъем ног, пресс, выпады, бурпи, прыжки со скакалкой, йога-поза "собака мордой вниз", бег на месте). Возможность просмотра описания/видео-примера для каждого.
  - Захват движений: Через фронтальную камеру (основная как альтернатива). Анализ позы с помощью машинного обучения.
  - Обратная связь: Реал-тайм индикаторы (зеленый — правильно, красный — ошибка) с текстовыми подсказками (например, "Выпрямите спину"). Подсчет повторений и сетов.
  - Дополнительно: Таймер для упражнения, пауза/стоп, сохранение результатов в статистику.
- **Требования**:
  - Офлайн-режим: Полная поддержка (модели ML загружаются локально).
  - Интеграция: Камера устройства (минимальное разрешение: 720p).
  - Ограничения: Точность анализа >80% в хорошем освещении; не поддерживать упражнения с оборудованием (гантели и т.д.); обработка ошибок (например, "Плохое освещение — попробуйте в другом месте").

##### 2.2. Функция 2: Диета и трекер питания
- **Описание**: Пользователь вводит антропометрические данные, приложение генерирует персонализированный план диеты и позволяет вести дневник потребления пищи с расчетом нутриентов.
- **Ключевой функционал**:
  - Ввод данных: Рост (см), текущий вес (кг), желаемый вес (кг), возраст, пол, уровень активности (по умолчанию: формула Mifflin-St Jeor для расчета суточной нормы калорий).
  - Генерация диеты: План на неделю (ежедневные калории с дефицитом/профицитом 500 ккал для потери/набора веса). Рекомендации по блюдам (по умолчанию: простые рецепты из базы, без учета аллергий).
  - Трекер питания: Ручной ввод продуктов (название, количество) с автоматическим расчетом калорий, белков, жиров, углеводов (БЖУ). Ежедневный журнал с итогами (сравнение с планом).
  - Дополнительно: Поиск по встроенной базе продуктов (по умолчанию: 500 популярных продуктов с данными из открытых источников, как USDA).
- **Требования**:
  - Офлайн-режим: Полная поддержка (база продуктов локальная).
  - Интеграция: Нет (по умолчанию; без штрих-кодов или внешних API в MVP).
  - Ограничения: Диета — ориентировочная, не медицинская; пользователь подтверждает, что данные верны; лимит ввода — 50 записей в день.

##### 2.3. Функция 3: Аккаунт и статистика
- **Описание**: Личный кабинет для авторизации, хранения данных и просмотра прогресса.
- **Ключевой функционал**:
  - Авторизация: Регистрация/вход через email, Google или Apple ID.
  - Статистика: Отслеживание метрик (дни подряд тренировок, потеря/набор веса, общие калории за неделю, прогресс по упражнениям — количество повторений/сетов).
  - Визуализация: Линейные графики (для веса/калорий), календарь (для тренировок), таблицы (ежедневные сводки).
  - Дополнительно: Экспорт данных (CSV), напоминания (локальные, без push в MVP).
- **Требования**:
  - Хранение данных: Локально (SQLite/Hive) с опцией облачной синхронизации (Firebase по умолчанию).
  - Офлайн-режим: Полная поддержка для просмотра (синхронизация при подключении).
  - Ограничения: Данные хранятся 1 год; лимит — 1 аккаунт на устройство.

#### 3. Нефункциональные требования
- **Дизайн и UI/UX**: Минималистичный стиль (светлая тема, простые цвета: зеленый для успеха, красный для ошибок). Адаптивный интерфейс для экранов 4-7 дюймов. Использовать Material Design (Android) / Human Interface Guidelines (iOS).
- **Производительность**: Время отклика интерфейса <1 секунда; анализ движений <2 секунды на кадр. Поддержка устройств с 2+ ГБ RAM.
- **Технологии**: 
  - Фреймворк: Flutter (кроссплатформенный).
  - ML для движений: ARKit (iOS), Google ML Kit (Android).
  - Хранение: Firebase (облако), Hive (локально).
  - Библиотеки: charts_flutter (графики), camera_flutter (камера).
- **Поддерживаемые версии ОС**: iOS 14+, Android 10+.
- **Безопасность**: Шифрование данных (AES), доступ к камере только с разрешения; соответствие GDPR (анонимизация данных).
- **Доступность**: Поддержка screen reader (VoiceOver/TalkBack), крупные шрифты.
- **Локализация**: Русский текст; метрики в кг/см.

#### 4. Интеграции
- Внутренние: Камера, хранилище устройства.
- Внешние: Firebase (auth, storage) — бесплатный tier для MVP.
- Дополнительно: Нет интеграций с устройствами (Fitbit и т.д.) или уведомлениями в MVP.

#### 5. Ограничения и допущения
- Приложение требует доступа к камере, хранилищу и интернету (для синхронизации).
- Офлайн-режим: 100% для основных функций, кроме синхронизации.
- Риски: Низкая точность ML в плохих условиях — добавить предупреждения.
- Допущения: Пользователь — взрослый; приложение не для медицинских целей (добавить disclaimer).
- Масштабируемость: MVP на 10k пользователей; для роста — облачный backend.

#### 6. Этапы разработки и тестирование
- **Этапы**: См. предыдущий план на 2 человек (подготовка, дизайн, разработка по спринтам, тестирование, деплой).
- **Тестирование**: Unit-тесты (80% покрытие), интеграционные, на реальных устройствах. Бета-тест на 10 пользователях.
- **Релиз**: Публикация в App Store и Google Play.
